Paint Playground

Paint is a collection of SCSS functions, mixins, placeholders and global styles that help us bootstrap our internal apps.

It also acts as an abstraction layer, keeping other frameworks / dependencies away from the applications. This makes structural changes (like removing a dependency or adding a new one) easier to implement.

Flex Grid

The main purpose of this helper component is to allow wrapping up any type of grids without having to copy paste the properties manually each time they are required.

The helper consists of two separate placeholders, %row and %col. The second is never going to work without having the first as a parent, so never extend `%col` without being in the context of a `%row`.

%row

Extending this is going to turn the child containers into columns. The `%row` also has control over the alignment of the items, through a set of modifiers:

  • Alignment

  • %row--middle

    Aligns all the columns vertically. Makes a difference only if the columns have different heights.

  • %row--center

    Aligns all the columns horizontally

  • %row--[top / bottom / left / right]

    Puts all the columns to the top / bottom / left or right

  • Positioning

  • %row--space-around

    Splits all the empty space around the columns, proportionally

  • %row--space-between

    Splits all the empty space proportionally between the columns (so the first and last columns will stick to the edges of the row)

  • Wrapping

  • %row--wrap

    Allows columns to go on multiple rows. All the alignment and positioning will be kept from line to line.

  • %row--vertical

    Turns columns into rows. This is still experimental, since we don't have any use cases yet.

  • Sizing

  • %row--fill

    Adds 100% width to the row. Useful with `position: absolute` or nested rows.

See the Pen Basic grid setup by Andrei Popa (@deioo) on CodePen.

%col

The `%col` consists of a set of placeholders to control the size / position / alignment of a single column element:

  • Alignment

  • %col--middle

    Aligns the current column vertically. Makes a difference only if the columns have different heights.

  • %col--center

    Aligns the column horizontally, pushing the columns from the left and right to the edges of the row.

  • %col--[top / bottom / left / right]

    Aligns the column to the top / bottom / left or right

  • Spacing

  • %col--gutter

    Adds a standard gutter margin to the column. The last-child in the row won't get a margin.

  • Sizing

  • %col-[1 - 12]

    Set the column width to a fixed grid size (1 of 12, 2 of 12 etc.). The total number of columns in a grid is 12.

  • %col-[push / pull]-[1 - 12]

    Pushes or pulls a column with x number of columns. (push adds the extra spacing to the left, pull adds the extra spacing to the right)

  • %col--fill

    Set the column to fill all the empty space of a row. If used on multiple rows, the space will be shared depending on each column content (so a column with more content will get more space than a column with less content)

  • %col--no-fill

    Resets the size of a column to grow as long as the content inside.

  • %col--auto-fill

    Resets both grow and shrink properties.

  • %col--no-shrink

    Makes sure the column won't shrink more than the width of the content inside. Useful e.g for avatars or fixed-size items, where we don't want the content to squash.

  • Responsive

  • %col-[small / medium / large / xlarge / xxlarge]-[1 - 12]

    Set the column width to a fixed grid size for a certain window size. Responsive column items can be combined to define different grid sizes for different resolutions.

  • %col-[small / medium / large / xlarge / xxlarge]-[push / pull]-[1 - 12]

    Pushes or pulls a column with x number of columns.

  • %col-[small / medium / large / xlarge / xxlarge]---fill

    Set the column to fill all the empty space of a row, in a responsive context. Useful if a column should fill small or medium screens, but follow a grid pattern for other sizes.
